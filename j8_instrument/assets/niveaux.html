Jeu de départ

Nous partons d’une base constituée d’un jeu de plateforme dans lequel un personnage est présent sur un monde présentant quelques plateaux. Il peut aller à gauche, à droite, et sauter avec la flèche du haut (voir Figure ci-contre). Cette base s’obtient facilement à partir du tutoriel de présentation de Phaser .

Comprendre le concept de scènes

Sur Phaser 3, on travaille avec des scènes. Globalement, un écran d’accueil, un niveau, une transition d’un mode à un autre, est défini par une scène.  Une scène contient trois fonctions principales, comme déjà vu :

  • La fonction preload() : charge des éléments (notamment des images) pour tout le jeu
  • La fonction create() : exécute des instructions lorsque la scène est lancée
  • La fonction update() : surveille les mises à jour, et événements de la scène

Dans les tutoriels simples, il n’y a qu’une scène, le code est simplifié. Mais si on veut mettre en place plusieurs scènes, il va falloir créer autant de fonctions preload, create et update que l’on a de scènes.  Pour charger plusieurs scènes dans Phaser 3, on va devoir modifier le code de manière conséquente en passant par une syntaxe « orientée objet ».

L‘utilisation des scènes est très simplifiée ici, mais elle  a l’avantage de marcher si les scènes sont très différentes, et est utilisée ici à des fins pédagogiques. Si on avait des niveaux très similaires, il serait possible de grouper toutes les scènes « au fonctionnement identique » en une seule, en faisant en sorte que les éléments différents soient mis à jour à chaque chargement de scène.

Changer la syntaxe de chargement de la scène 

Dans un premier temps, nous allons passer à une syntaxe orientée objets, en utilisant une classe. Avant, on avait quelque chose comme ça (les trois points signifient du code qu’on n’a pas copié pour des raisons de place) :

   var config = {
       ...
        scene: {
            preload: preload,
            create: create,
            update: update
        }
} ;

  var game = new Phaser.Game(config);

    function preload () { ... }
    function create ()  { ... }
    function update ()  { ... }

Il était sous-entendu qu’on avait qu’une seule scène, dont le nom n’est pas mentionné.  Pour passer à la syntaxe objet, il va nous falloir donner un nom à cette scène, par exemple « selectWorld», et modifier le code pour avoir quelque chose comme ceci : 

    window.onload = function() {
        var config = {
            ...
            scene: [selectWorld]
        };  
        game= new Phaser.Game(config);
        game.scene.start("selectWorld");
    }
    
 class selectWorld extends Phaser.Scene{

    constructor(){
         super({key: 'selectWorld' });
    }
    preload () { ...}
    create () { ...}
    update () { ...}
}

Qu’avons-nous fait ? nous avons regroupé les fonctions preload(), create()et update() au sein d’une classe du nom de la scène, ici avec la syntaxe class selectWorld. Cette classe doit hériter de la classe Phaser.Scene. Pour qu’elle fonctionne correctement,  il faut lui définir un constructeur – constructor()-, qui appele le constructeur de la classe mère – avec super()-  en lui passant en paramètre le nom de la classe comme clé – key : ‘selectWorld’- . Ceci est un peu technique, mais il suffit de comprendre qu’on a créé une scene nommée selectWorld, qui est associée à 3 méthodes :  preload(), create() et update(). 

On a du coup modifié légèrement la ligne de chargement de scene dans la variable config en précisant à l’intérieur la liste des scènes disponibles – ici juste [selectWorld]. Quand on a créé le jeu, on a ajouté juste apres le nom de la scène à lancer par défaut, avec la ligne game.scene.start(« selectWorld »);

Testons ce code : tout fonctionne. Afin d’éviter toute erreur de syntaxe, notez bien que :

  • la ligne scene de config a été remplacée, pas ajoutée
  • la fonction window.onload n’englobe pas tout, seulement les lignes de config et new Phase(game)
  • On a supprimé les mots clé ‘function‘ devant preload, create et update.

Ajout de portes vers 3 sous-niveaux

Nous allons ajouter trois nouvelles scènes, très similaires, nommées niveau1, niveau2 et niveau3, qui seront accessibles via 3 portes de couleurs différentes, et définies dans les images door1.png, door2.png et door3.png. Nous commençons par ajouter les portes dans la scène selectWorld de la façon suivante :

Hors de toute fonction, on va ajouter 3 variables door1, door2 et door3 :

var door1;
var door2;
var door3;

Dans la méthode preload() de selectWorld on charge les images en ajoutant :

      this.load.image('door1', 'assets/door1.png');
      this.load.image('door2', 'assets/door2.png');
      this.load.image('door3', 'assets/door3.png');

Dans la méthode create() de selectWorld on place les portes en ajoutant :

       door1 = this.add.sprite(600, 364,'door1'); 
       door2 = this.add.sprite(50, 214,'door2'); 
       door3 = this.add.sprite(750, 184,'door3');

Visuellement, nous avons le résultat suivant. les portes sont placées. Les prochaines étapes consistent à définir les 3 niveaux, puis à lier les portes aux niveaux.

Création des 3 sous-niveaux

Nous allons ajouter trois nouvelles scènes, très similaires, nommées niveau 1, niveau 2 et niveau3. Chaque niveau est une scène possédant ses propres fonctions preload, create, et upload. Il suffit de définir pour chaque scène une syntaxe similaire, en englobant nos trois fonctions dans une classe.  Nous présentons le code ajouté pour 1 niveau, les 2 autres étant globalement du copier coller (en changeant door1 par door2 et door3)  :

class niveau1 extends Phaser.Scene{
    constructor(){
         super({
            key: 'niveau1'
        });
    }
    preload () {
            this.load.image('sky', 'assets/sky.png');
            this.load.image('ground', 'assets/platform.png');
            this.load.image('door1', 'assets/door1.png');
            this.load.spritesheet('dude', 'assets/dude.png', { frameWidth: 32, frameHeight: 48 });
    }
     create () {
            this.add.image(400, 300, 'sky');
            platforms = this.physics.add.staticGroup();
            platforms.create(400, 568, 'ground').setScale(2).refreshBody();

            door1 = this.add.sprite(600, 516,'door1');
            player = this.physics.add.sprite(600, 516, 'dude');
            player.setBounce(0.2);
            player.setCollideWorldBounds(true);

           cursors = this.input.keyboard.createCursorKeys();
           this.physics.add.collider(player, platforms);
    }

   update () {
       console.log("niveau1");
        if (cursors.left.isDown)  {
            player.setVelocityX(-160);
            player.anims.play('left', true);
        }
        else if (cursors.right.isDown) {
            player.setVelocityX(160);
            player.anims.play('right', true);
        }
        else  {
            player.setVelocityX(0);
            player.anims.play('turn');
        }

        if (cursors.up.isDown && player.body.touching.down)  {
            player.setVelocityY(-330);
        }
    }
}

On fait la même chose avec niveau 2 et niveau 3. Il nous faut alors modifier le chargement de la configuration pour inclure ces 3 niveaux. Dans la variable config, onva remplacer la ligne scene: [selectWorld] par la ligne suivante : 

    scene: [selectWorld, niveau1, niveau2, niveau3]

On teste le jeu, rien n’a changé. Normal. Il reste à assurer les passages entre les différents niveaux. Pour cela, on va placer des portes reliant selectWorld à chacun des niveaux

Mise en place du mécanisme d’ouverture de porte

Nous ajoutons le mécanisme de porte suivant  : lorsque le joueur est sur une porte, il peut l’ouvrir en appuyant sur la touche espace, pas en laissant enfoncé, mais juste en appuyant dessus – D’un point de vue du développeur, on va surveiller quand le joueur appuie sur la touche espace, on va regarder s’il est dans la zone de la porte. Il y a une difficulté cachée dans le sens que les portes ne sont pas des objets sujets à la physique, mais de simples images posées à des coordonnées données. De ce fait on ne pourra pas utiliser directement la méthode add.overlap comme on le faisait avant pour des entités physiques. Mais pas de problème, on a une solution.  Hors de toute classe, on ajoute la fonction checkOverlap() qui renvoie vrai si 2 objets spriteA et spriteB passés en paramètre se superposent :

function checkOverlap(spriteA, spriteB) {
   var boundsA = spriteA.getBounds();
    var boundsB = spriteB.getBounds();
    var result = Phaser.Geom.Rectangle.Intersection(boundsA, boundsB);
    return (result.width !=0 || result.height!=0 )
}

Dans la méthode update() de selectWorld, on ajoute le mécanisme de surveillance de la touche espace. Si le joueur est sur une porte, on va lancer a scène correspondante :

        if (Phaser.Input.Keyboard.JustDown(cursors.space)) {
            if (checkOverlap(player, door1)) {
              this.scene.start("niveau1");
            } 
            if (checkOverlap(player, door2)) {
                this.scene.start("niveau2");
            } 
            if (checkOverlap(player, door3)) {
                this.scene.start("niveau3");
            }
        }

Au sein de chaque niveau, on peut adopter un mécanisme similaire : lorsque le joueur est dessus et appuie sur espace, on recharge le niveau de départ. Par exemple, pour le niveau 1, on écrira :

        if (Phaser.Input.Keyboard.JustDown(cursors.space) && checkOverlap(player, door1)) {
              this.scene.start("selectWorld");
        }

Pour les autres niveaux, on remplacera simplement door1 par door2, ou door3.

Il ne reste plus qu’a tester notre code et à admirer le résultat : quand on franchit une porte, en appuyant sur espace, on se retrouve dans un niveau. Chaque niveau se distingue par la couleur de la porte associée.

Pour aller plus loin …

Pour que le jeu soit plus agréable, il sera interessant de rajouter des mécanismes complémentaires. Par exemple si on revient sur le monde de départ, peut-être sera-t’il intéressant de sauvegarder dans une variable globale la porte qui a été franchie, pour positionner le personnage devant cette porte et non sur le point de départ de la scène.
